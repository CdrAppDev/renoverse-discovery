Discovery Summary

Here's what I took away from our conversation — let me know if I got it right.

What I Heard

Architects and builders working with state and municipal building codes need a way to get accurate, cited answers to compliance questions and rapidly identify what changed between code versions across multiple jurisdictions, because current manual approaches to code research and version comparison in several-hundred-page documents are slow, error-prone, and carry professional and legal risk when incorrect.

Who Will Use This

You described two groups of people who will use the platform:

Construction Practitioner (Architect or Builder)

A licensed architect or builder who must interpret and apply building code requirements to specific project decisions. They read state and municipal building codes to determine how a project must be designed or constructed to meet regulatory requirements — and must stay current as those codes are updated every three to six months.

Platform Administrator

A person responsible for maintaining the platform's document library — acquiring new code versions when they are published, loading them into the system, and ensuring users aligned with the relevant jurisdiction are notified of the update. Administrators manage documents at the state, county, and city level and are responsible for keeping the platform's content current.

What They Need to Do

The core problem your platform solves:

Determine how to meet building code requirements for a specific construction decision

Your users also need to:

  • Stay current with building code changes affecting active projects The version comparison feature directly serves this need — practitioners need to know which requirements changed in a new code version and whether those changes affect work already under design or construction.
  • Verify a specific compliance claim against the authoritative code text Practitioners often need to verify whether a claim — from a colleague, from memory, or from a prior project — is actually supported by the current code.
  • Identify which jurisdiction's code governs a specific project element When state, county, and city codes all apply, practitioners must determine which authority governs a specific requirement before they can look up the answer.

Future Considerations

These came up in my analysis but aren't part of what you described. They might be worth thinking about down the road:

  • Assess impact of code changes on active projects — When a new code version is adopted, determining which projects already under design are affected.
  • Document compliance decisions for the project record — Recording which version and section was relied upon for each compliance decision.

  • Resolve inspector disputes — Defending a compliance position when there's a disagreement about code interpretation.
  • Firm principal or office manager as a distinct user type — In team or firm contexts, the person who decides to adopt the platform may have distinct needs around administration and oversight.
